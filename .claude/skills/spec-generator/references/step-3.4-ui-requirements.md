# Step 3.4: UI Requirements 詳細化

spec.md の UI Requirements セクションは、scaffold-fe が推測せずに実装できるレベルまで詳細化する。
`.claude/references/design-system.md` を参照してトークン・パターンを正しく使用する。

---

## 必須サブセクション

| サブセクション                 | 内容                         | 省略条件                               |
| ------------------------------ | ---------------------------- | -------------------------------------- |
| コンポーネント階層             | DOM ツリー構造               | 画面がない場合のみ省略可               |
| レイアウト・スペーシング       | 配置・幅・余白の具体的な値   | 画面がない場合のみ省略可               |
| 状態ごとの表示ルール           | 条件 × 要素のマトリクス      | 状態変化がない場合のみ省略可           |
| インタラクション               | 操作と結果の対応             | 必須                                   |
| アニメーション・トランジション | トリガーとアニメーション種別 | アニメーションがない場合「なし」と明記 |
| レスポンシブ挙動               | ブレークポイントごとの変更   | モバイル専用の場合「なし」と明記       |
| バリデーション表示             | エラーメッセージの表示ルール | フォームがない場合のみ省略可           |
| 空状態・ローディング           | 各状態の表示内容             | データ一覧画面のみ必須                 |

---

## 状態ごとの表示ルールの書き方

scaffold-fe が推測しなくて済むよう、**条件 → 表示 → 具体的なクラス**まで書く。

### NG 例（scaffold-fe が推測する余地が残る）

```
- 他ユーザーのデータは閲覧のみ
```

### OK 例

```
| 要素 | 条件 | 表示 | 備考 |
|------|------|------|------|
| EditButton | 自分のデータ & status=editable | 表示 | |
| EditButton | 他人のデータ | 非表示（DOM除去） | |
| DeleteButton | status=completed | disabled（opacity-50, cursor-not-allowed） | tooltip「完了済みは削除できません」 |
```

---

## アニメーションの書き方

design-system.md の方針「アニメーション過多を避ける」に従い、必要最小限のみ定義。
ダイアログ開閉、リスト要素の出入り、トースト通知など、ユーザー体験に直結するものだけを記述する。

---

## レスポンシブの書き方

Tailwind のブレークポイント（sm/md/lg）を基準に、各サイズでのレイアウト変更を明記する。
変更がない行も「変更なし」と明記する。
