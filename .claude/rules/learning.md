# Learning Rules

## 継続的改善プロセス

### ルール追加のタイミング

Claude がミスを犯した場合、以下のように CLAUDE.md へのルール追加を**提案**する。
人間がレビュー・承認した後にルールを追加する:

```markdown
## Rule: [ルール名]

- 何をすべきか/すべきでないか
- なぜそうすべきか（理由）
- 具体例
```

### 追加すべきルールの例

| 状況                             | 追加するルール             |
| -------------------------------- | -------------------------- |
| 同じミスを2回以上した            | そのミスを防ぐルール       |
| 非効率なアプローチを取った       | より良いアプローチの説明   |
| プロジェクト固有の慣習を無視した | その慣習の明文化           |
| 外部 API の使い方を間違えた      | 正しい使い方のドキュメント |

### ルール記載のベストプラクティス

1. **具体的に**: 「良いコードを書く」ではなく「エラーハンドリングには Result パターンを使う」
2. **理由を含める**: なぜそのルールが必要か
3. **例を示す**: 良い例と悪い例の両方
4. **簡潔に**: 1 ルール = 数行

### メンテナンス

- 週に数回 CLAUDE.md をレビュー
- 古くなったルールは更新または削除
- 重複するルールは統合

## Auto Memory との連携

Claude Code の auto memory（`.claude/projects/*/memory/MEMORY.md`）は、セッション間で知見を自動蓄積する仕組み。
CLAUDE.md のルールと組み合わせて2段階で知見を管理する。

### 役割分担

| ドキュメント                   | 役割                         | 更新タイミング     |
| ------------------------------ | ---------------------------- | ------------------ |
| `MEMORY.md`                    | 試行錯誤の記録、一時的な学び | セッション中に自動 |
| `CLAUDE.md` / `.claude/rules/` | 安定したルール、チーム共通   | 人間のレビュー後   |

### 推奨フロー

1. 開発中にミスや知見が発生 → `MEMORY.md` に自動記録
2. 同じパターンが複数回出現 → CLAUDE.md へのルール追加を提案
3. 人間がレビュー・承認 → `.claude/rules/` にルール追加
